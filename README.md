## Описание проекта

Реализовано backend-приложение на Django + DRF.

Проект представляет собой API, через которое можно:
- получать список товаров,
- добавлять товары в корзину,
- оформлять заказ.

Основной упор был на:
- понимание, как устроен Django (ORM, модели, связи),
- работу с DRF (сериализаторы, APIView),
- организацию структуры проекта.

## Реализованный функционал

### Каталог товаров

- `GET /products/` - получение списка товаров;
- фильтрация:
  - `search` - поиск по названию;
  - `category` - фильтр по категории;
- `GET /products/<id>/` - получение информации о товаре;
- отображаются только активные товары (`is_active=True`);
- поддержка пагинации.

------------------------------------------------------------------------

### Корзина

- `GET /cart/` - получение корзины пользователя;
- автоматическое создание корзины при первом обращении.
- `POST /cart/add/` - добавление товара в корзину;
  - увеличение количества при повторном добавлении;


------------------------------------------------------------------------

### Заказы

- `POST /orders/create/` - создание заказа из корзины;
  - проверка наличия товара на складе;
  - перенос позиций из корзины в заказ;
  - фиксация цены на момент покупки;
  - очистка корзины после оформления;
- `GET /orders/` - получение списка заказов пользователя;
- `GET /orders/<id>/` - получение конкретного заказа.
  - реализован доступ только к заказам текущего пользователя.

## Описание архитектуры

Проект построен по классической архитектуре Django + DRF:

- **models.py** - описание структуры данных и связей;

## Модели данных

### Category

Категория товаров.
Поля: 
- `name` - название;
- `slug` - уникальное имя.

### Product

Товар, связан с категорией.
Поля:
- `category` - категория;
- `name` - название;
- `slug` - уникальное имя;
- `description` - описание;
- `price` - цена;
- `stock` - остаток на складе;
- `is_active` - активен/скрыт;
- `created_at` - дата создания.


### Cart

Корзина пользователя. Поля:
- `user` - пользователь (OneToOne).
- `created_at` - дата создания.


### CartItem

Позиция в корзине. Поля:
- `cart` - корзина;
- `product` - товар;
- `quantity` - количество единиц товара.

Обеспечена уникальность пар (`cart`, `product`).

### Order

Заказ пользователя.
- `user` - пользователь;
- `created_at` - дата создания;
- `status` - статус (один из трех вариантов).


### OrderItem

Позиция заказа. Поля:
- `order` - заказ;
- `product` - товар;
- `quantity` - количество единиц товара;
- `price` - цена при заказе. Фиксируется на момент покупки
